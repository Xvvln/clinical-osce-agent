# 学生端阑尾炎病例演示脚本

本文档用于演示学生端如何围绕“右下腹痛教学病例”完成一次 OSCE 训练，并展示系统如何从训练表现进入报告复盘、来源追溯和后续训练改进。

注意：本文档只作为演示手册，不写入病例 JSON、Rubric、模型配置或运行时数据库。项目页面中的输入框、API Key、服务端地址、诊断提交内容仍保持由演示者现场填写。

## 演示目标

- 展示学生如何从病例选择进入完整训练流程。
- 展示问诊、查体、辅助检查、诊断提交和评分报告的连续闭环。
- 展示评分报告中的结构化来源引用和学习建议。
- 展示学生犯错后，系统如何把漏项转化为下一轮训练重点。
- 如管理员端已审核启用相关 Skill，可展示下一轮训练中 `/hint` 或反馈如何体现 Skill 生效。

## 推荐病例

主病例使用 `appendicitis_001`，页面标题为“右下腹痛教学病例”。

推荐理由：

- 当前病例信息最完整，适合展示问诊、查体、辅助检查、诊断推理和鉴别诊断。
- 病例主题是急腹症入门场景，非医学评审也容易理解。
- 容易构造高频错误模式，例如漏问疼痛转移、漏查反跳痛、缺少鉴别诊断、诊断正确但证据链表达不足。

## 演示前准备

1. 启动后端和学生端前端。
2. 在学生端登录测试账号。
3. 打开导航球中的 API 配置，只在演示现场填写需要的服务端或模型配置。
4. 如果使用 Vertex Gemini ADC，API Key 保持空白，选择 Vertex Gemini ADC 并填写本机可用的项目编号或项目 ID。
5. 点击连通性测试，确认配置可用后关闭弹窗。
6. 进入病例库，选择“右下腹痛教学病例”。

## 第一轮训练：故意暴露典型漏项

这一轮不要表现得太完美，目的是让系统产生可解释的训练报告和错误模式。

### 1. 进入病例

在病例库点击“开始训练”或回到训练工作台后选择“右下腹痛教学病例”。

演示时强调：学生只能看到主诉和训练任务，看不到隐藏事实、标准诊断、Rubric 和完整来源台账。

### 2. 开始问诊

建议按以下顺序输入，故意漏掉部分关键问题：

```text
你哪里不舒服？
```

```text
疼痛大概有多严重？
```

```text
有没有恶心、呕吐或者发烧？
```

这一轮先不要追问“疼痛有没有转移”，也不要追问“大便、小便情况”。这样报告里会稳定出现问诊漏项。

### 3. 请求过程提示

点击“请求提示”。

观察点：

- 如果没有启用 Skill，提示会基于当前训练进度引导下一步。
- 如果管理员端已经审核启用了相关 Skill，提示应体现“本轮训练重点”，例如引导学生补齐腹痛演变史或证据链。
- 训练日志中后续可以看到 `hint_requested`，如果 Skill 生效，还应能看到 `training_skill_applied`。

### 4. 查体

先只选择或输入一个不完整查体：

```text
测量体温
```

然后补一个关键查体：

```text
检查右下腹压痛
```

这一轮可以故意不查反跳痛或肌紧张，让报告产生“腹膜刺激征检查不足”的反馈。

### 5. 辅助检查

申请基础检查：

```text
血常规
```

```text
腹部超声
```

这一轮可以故意不申请尿常规，让系统在鉴别诊断部分提示“排除输尿管结石证据不足”。

### 6. 提交诊断

在最终诊断输入框填写：

```text
急性阑尾炎
```

推理说明可以故意写得偏短：

```text
患者右下腹痛，有发热和白细胞升高，所以考虑急性阑尾炎。
```

观察点：

- 主诊断通常能命中。
- 推理说明会因为缺少“转移性疼痛、反跳痛、影像学证据、鉴别诊断排除”而被指出不完整。
- 这能体现系统不是只看诊断是否正确，而是评价完整临床思维链。

## 查看评分报告

提交诊断后打开评分报告页。

重点展示以下区域：

- 总分与各维度得分。
- 问诊漏项，例如疼痛转移特征、相关阴性症状。
- 查体漏项，例如反跳痛或腹膜刺激征。
- 辅助检查漏项，例如尿常规用于排除输尿管结石。
- 推理反馈，例如诊断证据链不完整。
- 下一轮训练建议。
- 来源引用区域，包括 `source_references`、`source_reference_items` 和 `explanation_source_items`。

演示话术可以强调：

```text
这里不是让大模型随意点评学生，而是把报告中的解释、建议和漏项追溯到病例结构化事实、Rubric 条目、训练证据和来源台账。
```

## 第二轮训练：展示改进

重新进入同一病例，完成第二轮训练。

这次按系统建议补齐关键步骤。

### 1. 补齐问诊

```text
疼痛一开始在哪里，现在有没有位置变化？
```

```text
疼痛是什么性质，走路或咳嗽会不会加重？
```

```text
有没有腹泻、大便异常、尿频尿急或者血尿？
```

### 2. 补齐查体

```text
检查 McBurney 点压痛
```

```text
检查反跳痛
```

```text
检查肌紧张和 Rovsing 征
```

### 3. 补齐辅助检查

```text
血常规
```

```text
C 反应蛋白
```

```text
腹部超声
```

```text
尿常规
```

### 4. 提交更完整推理

最终诊断：

```text
急性阑尾炎
```

推理说明：

```text
患者先出现上腹部隐痛，随后转移并固定在右下腹，伴恶心和低热。查体有 McBurney 点压痛、反跳痛和局部腹膜刺激征，血常规白细胞及中性粒比例升高，CRP 升高，腹部超声提示阑尾增粗并有周围渗出，因此支持急性阑尾炎。尿常规阴性且无明显血尿，可帮助排除右侧输尿管结石；病程急性，也不支持克罗恩病回盲部受累。
```

观察点：

- 问诊覆盖度提高。
- 查体和辅助检查更完整。
- 推理反馈应明显更好。
- 下一轮建议会从基础漏项转向更高层次的鉴别诊断或证据整合。

## 可选：展示 Skill 自学习闭环

如果管理员端已经从训练日志生成并审核通过候选 Skill，可以继续演示：

1. 管理端打开训练洞察，查看高频漏项或错误模式。
2. 点击“从训练日志生成候选 Skill”。
3. 查看候选 Skill 详情，确认它面向整体错误模式，而不是单个 missed item。
4. 审核通过候选 Skill。
5. 回到学生端重新开始同一病例。
6. 点击“请求提示”或完成训练后查看反馈。
7. 展示提示或反馈中出现已启用 Skill 的训练策略。
8. 管理端查看 Skill 应用次数；如果样本不足，应显示“样本不足”，不展示伪造提升。

推荐候选 Skill 主题：

- 腹痛演变史采集训练。
- 腹膜刺激征检查完整性训练。
- 急腹症证据链整合训练。
- 右下腹痛鉴别诊断排除训练。

## 不建议演示的做法

- 不要把标准答案、隐藏事实或完整推理提前写在学生端输入框里。
- 不要在病例 JSON 或 Rubric 中硬编码演示专用文本。
- 不要让 RAG 参与诊断裁判或评分决策。
- 不要伪造 Skill 效果提升；样本不足时就展示样本不足。
- 不要把 ACS 胸痛病例作为第一主线，避免答辩焦点滑向真实急救处置和用药细节。

## 演示结束后检查

- 训练记录中能看到两轮 session。
- 每轮 session 都能打开评分报告。
- 报告中能看到来源引用。
- 学习画像能体现最近训练和薄弱方向。
- 如果启用了 Skill，管理端能看到应用记录或样本不足提示。
- API 配置仍由运行时或浏览器本地配置控制，项目文件中没有写入个人密钥或演示填充值。

# 数据来源说明

## 已接入或已登记的数据 / 参考源

### 1. Fareez OSCE 数据集
- 来源：Figshare / Scientific Data
- 本地策略：下载原始 `Data.zip` 到 `data/raw/fareez_osce/`
- 用途：问诊语料风格、话轮结构、症状采集模式参考
- 许可：CC BY 4.0

### 2. MediTOD
- 来源：GitHub 公共仓库
- 本地策略：克隆到 `references/MediTOD/`
- 用途：意图识别、槽位抽取、对话标注结构参考
- 限制：canonicalized 数据需要 UMLS 许可和额外申请，不自动下载

### 3. MedCaseReasoning
- 来源：Hugging Face 公共数据集
- 本地策略：下载公开 parquet 文件到 `data/raw/medcase_reasoning/`
- 用途：诊断推理依据、鉴别诊断 reasoning point 提炼

### 4. CaseReportCollective
- 来源：Hugging Face 公共数据集
- 本地策略：下载公开 parquet 文件到 `data/raw/case_report_collective/`
- 用途：二期扩展病例来源，不直接作为 MVP 主数据

### 5. 参考工程
- `references/agent-chat-ui/`：前端交互参考
- `references/agent-service-toolkit/`：后端分层与服务结构参考
- `references/EasyMED/`：医学教育多智能体模块参考

## Git 提交策略

以下内容本地保留，不提交到 GitHub：

- `references/`
- `data/raw/`

这样可以避免：

1. 外部仓库嵌套导致 Git 历史混乱；
2. 大体积原始数据推高仓库体积；
3. 后续清洗与结构化结果难以区分。
